## 문제 1) 비정규화(반정규화, 역정규화) 개념 간략 설명

**문제**:  
비정규화(반정규화, 역정규화)의 개념을 간략히 설명하시오.

**정답**:  
성능 향상과 개발 편의성 등을 위해 정규화 기법에 위배되는 수행 기법

**관련 설계 과정 요약**:  
- 개념적 설계: ERD  
- 논리적 설계: 목표 DBMS, 트랜잭션 인터페이스(T/I), 정규화  
- 물리적 설계: 성능 중심 → 파티션, 클러스터, 인덱스 등

**정규화 과정 요약**:  
- 1정규형: 도메인이 원자값  
- 2정규형: 부분 함수 종속 제거  
- 3정규형: 이행 함수 종속 제거  
- BCNF: 결정자이면서 후보키가 아닌 것 제거  
- 4정규형: 다치 종속 제거  
- 5정규형: 조인 종속 제거

## 문제 3) 살충제 패러독스

**문제**:  
살충제 패러독스 개념을 간략히 설명하시오.

**정답**:  
동일한 테스트 케이스로 동일한 절차를 반복 수행하면 새로운 결함을 찾을 수 없다.

**관련 테스트 개념**:

- **소프트웨어 테스트 단계**  
  - 단위 테스트:  
    - 정적 테스트: 소스코드 보면서  
    - 동적 테스트: 소스코드 실행  
  - 통합 테스트:  
    - 하향식: 스텁 사용  
    - 상향식: 드라이버 사용  
    - 빅뱅 테스트: 비점진적  
  - 시스템 테스트:  
    - 기능 테스트 / 비기능 테스트  
  - 인수 테스트:  
    - 알파 테스트(개발자 있음) / 베타 테스트(개발자 없음)

- **테스트 관련 법칙 및 개념**  
  - 파레토 법칙: 20% 모듈이 80% 오류  
  - 테스트 오라클: 참값 대입  
    - 참 오라클 / 샘플 오라클 / 휴리스틱 오라클 / 일관성 검사 오라클

## 문제 4) 모듈의 독립성

**문제**:  
모듈의 독립성을 판단하는 두 가지 지표이다.  
(1)은 모듈 간 상호 의존 정도, (2)는 모듈 내부의 기능적 집중 정도이다.  
모듈의 독립성을 높이기 위해서는 (1)을 낮추고 (2)를 높여야 한다.

**정답**:  
(1) 결합도(Coupling), (2) 응집도(Cohesion)

**응집도 (높을수록 좋음)**:  
- 우연적 < 논리적 < 시간적 < 절차적 < 통신적 < 순차적 < 기능적  

**결합도 (낮을수록 좋음)**:  
- 내용 > 공통 > 외부 > 제어 > 스탬프 > 데이터  

**관련 개념**:  
- 재공학: 기존 시스템 재활용  
- 역공학: 소스코드로 문서 추출  
- 재사용: 모듈 재활용
