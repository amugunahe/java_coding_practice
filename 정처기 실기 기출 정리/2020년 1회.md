## ✅ 문제 1) 비정규화(반정규화, 역정규화) 개념 간략 설명

- **답**: 성능 향상  
- 성능 향상과 개발 편의성 등을 위해 정규화 기법에 위배되는 수행 기법

### 📌 설계 단계별 구성
- **개념적 설계**: ERD
- **논리적 설계**: 목표 DBMS, T/I(트랜잭션 인터페이스), 정규화
- **물리적 설계**: 성능 → 파티션, 클러스터, 인덱스 등

### 📌 정규화
- **1정규형**: 도메인이 원자값
- **2정규형**: 부분 함수 종속 제거
- **3정규형**: 이행 함수 종속 제거
- **BCNF**: 결정자이면서 후보키가 아닌 것 제거
- **4정규형**: 다치 종속 제거
- **5정규형**: 조인 종속 제거

## ✅ 문제 12) 트랜잭션 특성 4가지

- **답**
  - (1) 원자성(Atomicity)
  - (2) 격리성(Isolation)

### 📌 트랜잭션 특성 (ACID)
- **Atomicity (원자성)**: 전부 수행 or 전부 미수행

- **Isolation (격리성)**: 트랜잭션 간 간섭 X
- **Durability (영속성)**: 성공 결과는 영구 저장됨

## ✅ 문제 16) 해시 알고리즘 – MD5

- **답**: MD5

### 📌 설명
- 1990년 R. Rivest가 개발
- MD4를 개선한 해시 알고리즘
- 임의의 길이 입력 → 128비트 해시 출력

### 📌 암호화 종류

#### 🔐 대칭키 알고리즘
- 장점: 속도 빠름, 많은 양 처리
- 단점: 키 분배 문제
- 알고리즘: DES, AES, ARIA, SEED, IDEA, LEA, SKIPJACK

#### 🔐 비대칭키 알고리즘
- 장점: 키 분배 문제 해결
- 단점: 속도 느림, 적은 양에 사용
- 종류:
  - 소인수 분해 기반: RSA, Rabin
  - 이산대수 기반: Diffie-Hellman, DSA, ElGamal
  - 타원곡선 기반: ECC

#### 🧩 해시 알고리즘
- SHA, MD5, HAS-160

#### 📌 해시 취약점
- 레인보우 테이블 → 솔팅(Salting) 필요
- 무차별 공격 → 키 스트레칭(Key Stretching) 필요

#### 📌 해시 속성
- 역상 저항성
- 제2 역상 저항성
- 충돌 저항성
