## ✅ 문제 1) 비정규화(반정규화, 역정규화)

- **문제**: 비정규화(반정규화, 역정규화)의 개념을 간략히 설명하시오.
- **답**: 성능 향상과 개발 편의성 등을 위해 정규화 기법에 위배되도록 수행하는 기법

### 🔎 설계 단계별 특징

- **개념적 설계**: ERD 작성
- **논리적 설계**: 목표 DBMS 설정, 트랜잭션/인터페이스 설계, 정규화 수행
- **물리적 설계**: 성능 중심 설계 (파티션, 클러스터, 인덱스 등)

### 🔍 정규화 종류

| 정규형 | 설명 |
|--------|------|
| 1NF | 도메인이 원자값 |
| 2NF | 부분 함수 종속 제거 |
| 3NF | 이행 함수 종속 제거 |
| BCNF | 결정자이면서 후보키가 아닌 속성 제거 |
| 4NF | 다치 종속 제거 |
| 5NF | 조인 종속 제거 |

## ✅ 문제 3) 살충제 패러독스

- **문제**: 살충제 패러독스 개념을 간략히 설명하시오.
- **답**: 동일한 테스트 케이스로 동일한 절차를 반복 수행하면 새로운 결함을 찾을 수 없다.

### 🔍 소프트웨어 테스트 유형

- **단위 테스트**
  - 정적 테스트: 소스코드만 분석
  - 동적 테스트: 실행하면서 테스트
- **통합 테스트**
  - 하향식: 스텁 사용
  - 상향식: 드라이버 사용
  - 빅뱅 테스트: 비점진적 테스트
- **시스템 테스트**
  - 기능 테스트, 비기능 테스트
- **인수 테스트**
  - 알파 테스트: 개발자 있음
  - 베타 테스트: 개발자 없음

### 기타 개념

- **파레토 법칙**: 20% 핵심 모듈에서 80% 오류 발생
- **테스트 오라클 종류**:
  - 참 오라클
  - 샘플 오라클
  - 휴리스틱 오라클
  - 일관성 검사 오라클

## ✅ 문제 4) 결합도와 응집도

- **문제**: 모듈의 독립성을 판단하는 두 가지 지표이다.  
  (1)은 모듈과 모듈 간의 상호의존 정도,  
  (2)는 모듈 내부의 기능적 집중 정도이다.  
  모듈의 독립성을 높이기 위해 (1)을 낮추고 (2)를 높여야 한다.

- **답**: (1) 결합도, (2) 응집도

### 🔍 응집도 (높을수록 좋음)

| 유형 | 설명 |
|------|------|
| 우연적 | 가장 낮음. 아무 의미 없이 모아둔 모듈 |
| 논리적 | 유사한 기능을 모아둔 모듈 |
| 시간적 | 동일 시간에 실행되는 기능 모음 |
| 절차적 | 순차적으로 실행되는 기능 |
| 통신적 | 동일 입력/출력 자료를 사용하는 기능 모음 |
| 순차적 | 앞 모듈 결과가 뒷 모듈 입력이 되는 형태 |
| 기능적 | 가장 높음. 하나의 기능에 집중된 모듈 |

### 🔍 결합도 (낮을수록 좋음)

| 유형 | 설명 |
|------|------|
| 데이터 | 값만 전달 |
| 스탬프 | 구조체/배열 전달 |
| 제어 | 제어 변수 전달 |
| 외부 | 외부 변수 사용 |
| 공통 | 전역 변수 사용 |
| 내용 | 가장 높음. 내부 로직 간섭

## ✅ 문제 9) Land Attack

- **문제**:
  - 출발지 IP와 목적지 IP가 같은 패킷을 만들어 전송
  - 수신자가 응답을 보낼 때 자기 자신에게 전송되어 자원 고갈
  - 방화벽에서 출발지/목적지가 같은 패킷을 제거하여 대응

- **답**: Land Attack

### 🔍 기타 공격 유형

- **Smurf Attack**: ICMP로 브로드캐스트
- **SYN Flooding**: 과도한 SYN으로 자원 고갈
- **Ping of Death**: 비정상적으로 큰 패킷
- **UDP Flooding / Ping Flooding / Teardrop Attack**

## ✅ 문제 12) 트랜잭션의 4가지 특성

- **문제**: 다음 설명에 해당하는 트랜잭션 특성을 쓰시오.
  - (1): 모든 연산이 완전히 수행되거나 전혀 수행되지 않아야 함
  - (2): 다른 트랜잭션과 격리되어야 함

- **답**:
  - (1) 원자성(Atomicity)
  - (2) 격리성(Isolation)

### 💡 나머지 특성

- **일관성(Consistency)**: 정합성 유지
- **지속성(Durability)**: 시스템 장애 후에도 결과 유지
